From my understanding

  • vec2(x, y) — In this case this 2d vector is used to represent each vertex position on the canvas. The center of the circle is a vec2(0, 0), and every rim point is also a vec2 computed from cos and sin.
  • vec3(r, g, b) — The center gets white vec3(1,1,1) and each rim vertex gets a different hue-derived color.
  • hsvToRgb(h, s, v) — Converts a color from Hue, Saturation, Value space to RGB. So we give it a value hue in [0, 1] range to returns an array (r, g, b). We can use it to go through a full rainbow .
  • flattenArrays(points, colors) — Interleaves the two arrays into one flat buffer per vertex: [x, y, r, g, b, x, y, r, g, b, etc]. Both arrays must be the same length, which is why every points.push() is paired with a colors.push().

2a — Circle geometry and color interpolation

To build the circle I first push a center vertex at (0, 0) with white color, which acts as the anchor for all the triangle slices. Then I loop segments until it closes the circle so the outline connects back to the start seamlessly.

For the colors, I use the hue value to get a different color for each rim vertex, creating a rainbow effect. The hue is computed as i / segments so it goes from 0 to 1 as we go around the circle.

2b — Triangle fan fill and line-strip outline

Using an index buffer with the pattern [0,1,2], [0,2,3], [0,3,4] where vertex 0 is always the center of our circle to share between all slices and i, i+1 are two consecutive rim points. The fill pipeline uses triangle-list topology and drawIndexed so the GPU looks up which vertices to use for each slice, letting vertex 0 be reused every time without duplicating it in the buffer.

For the black outline using a separate vertex buffer that contains only the rim vertices to set them black [0,0,0]. The outline pipeline uses line-strip topology, which connects each consecutive vertex to the next, tracing the edge of the circle. Drawing it after the fill means it always appears on top.

2c — Why does the circle appear non-round?

From what I understand, the vertex coordinates are in Normalized Device Coordinates (NDC), where x and y both range from -1 to +1 regardless of the actual canvas size. The GPU then maps x across the full canvas width and y across the full height. Since our canvas is 800 × 600 and not square, the scaling isn’t equal, so a circle in NDC gets stretched wider than it is tall, making it look like a horizontal ellipse. To fix this, we apply an aspect ratio correction in the vertex shader, usually by dividing the x coordinate by (width / height) so both axes scale evenly.
