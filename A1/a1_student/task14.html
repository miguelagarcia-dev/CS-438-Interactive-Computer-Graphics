Task 4-1 – UV Sphere Geometry

For this I used spherical coordinates to place vertices on a unit sphere. Each vertex gets two angles: theta going top to bottom (0 at the north pole to PI at the south pole) and phi sweeping all the way around (0 to 2*PI). Those convert to x, y, z using the standard spherical to cartesian formulas with y as the up axis. The grid is (heightSegments+1) by (widthSegments+1) so the seam and poles close up without any gaps. Each quad gets split into two CCW triangles. The noise color comes from simplex3 sampled at the 3D position, shifted from [-1, 1] to [0, 1], and pushed into all 3 color channels so it comes out grayscale.

Task 4-2 – Earth Model Matrix

The way I set this up, each mult call wraps the next transform on the outside, so reading the calls from bottom to top gives you the actual order a point goes through: scale first, then spin on its axis (dayDeg), then tilt -23.44 degrees around Z, then translate out to orbit distance, then orbit the Sun (yearDeg). The tilt goes between the axial spin and the translation so it stays constant in the x-y plane and doesn't rotate along with the yearly orbit. Combined with viewMatrix at the end.

Task 4-3 – Moon Model Matrix

The Moon has to follow everything Earth does and then add its own orbit on top. So the chain starts the same as Earth (orbit around Sun + jump to Earth's position), then adds the Moon's own rotation around Earth (monthDeg), its distance offset from Earth, its -5.14 degree tilt, and finally the scale. Since the Moon's local transforms are inside Earth's orbit transform, it automatically tags along with Earth while still going around it.

Task 4-4 – Perlin Noise Shading

noiseColor comes in as grayscale so all 3 channels have the same value - I just use x as a scalar n. I multiply the base material color by (0.4 + 0.6 * n) so the noise darkens some spots without making anything fully black. It ends up looking kinda like rough terrain or cloud patches on the surface. Alpha stays at u_material.w.
